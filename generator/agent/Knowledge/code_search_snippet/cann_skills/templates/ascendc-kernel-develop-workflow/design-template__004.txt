Reduce 操作?
    │
    ├─ 数据布局？
    │    ├─ AR 模板 (A0=1, 每行 R 个连续元素)
    │    │    └─→ ✅ Level 2 API (推荐)
    │    │             ReduceMax(dst, src, tmp, count)
    │    │             - 无对齐要求
    │    │             - 输出: 标量(1 个值)
    │    │             - 性能最优
    │    │
    │    └─ ARA 模板 (A0>1, R×A0 数据, 归约 R 维需 stride)
    │         └─→ ⚠️ Pattern API (必须)
    │              ReduceMax<T, Pattern::RA>(dst, src, tmp, srcShape, true)
    │              - srcShape[1] 必须 32 字节对齐
    │              - 输出: 向量(A0 个值)
    │              - ⚠️ **禁止使用 Level 2 API**
    │                 （Level 2 无法处理 stride，会导致错误）
    │
    └─ ❌ 禁止凭直觉选择 API
              必须参考此决策树！
