We need to generate the six Python strings for the `elu_custom` kernel (PascalCase: `EluCustom`). The example for leaky_relu_custom gives a template. We need to adapt it for ELU.

ELU formula: y = x if x > 0; else alpha * (exp(x) - 1). We need to implement this efficiently in Ascend C.

Key points:
- Input: float x, float alpha (attribute).
- Output: float y same shape.
- Use tiling, UB budget similar to LeakyRelu.

We need to generate:
1. `project_json_src` - operator definition.
2. `host_tiling_src` - tiling data structure.
3. `host_operator_src` - tiling function, infer shape, op registration.
4. `kernel_src` - kernel implementation.
5. `python_bind_src` - pybind and torch custom op.
6. `model_src` - model using the custom op.

We must be careful with naming: kernel function `elu_custom`, PascalCase `EluCustom`.

In kernel, we need to compute exp(x) for negative x. Ascend C provides `Exp` intrinsic. We'll use local tensors, pipeline.

Let's design kernel algorithm:
- For each element, compute if x > 0 then y = x else y = alpha * (exp(x) - 1).
- We can compute t = min(x, 0) to get negative part, then compute exp of that? But exp(x) for negative is fine. Alternatively, use vectorized operations:
  - Compute mask: where x > 0. But Ascend C doesn't have direct conditional assignment in vector mode? We can use `Muls` and `Add` with mask? Simpler: compute two branches and blend.
  - Use `Mins` to get negative part: neg_part = min(x, 0). Then for negative part we need to compute exp(neg_part). But exp(0) = 1, so for x>0, neg_part=0, then alpha*(exp(0)-1)=0, which is correct (since y=x for x>0, we later need to add positive part). Actually we can compute: y = x + alpha * (exp(min(x,0)) - 1). For x>0, min=0 => exp(0)-1=0 => y=x. For x<0, y = x + alpha*(exp(x)-1) = alpha*(exp(x)-1) + x. But correct ELU is alpha*(exp(x)-1) for x<=0. Wait: For x<=0, y = alpha*(exp(x)-1). Our formula gives y = x + alpha*(exp(x)-1) which is wrong. So we need to subtract x? Actually we want to replace x with alpha*(exp(x)-1) when x<=0. So we can compute: neg_part = min(x,0); then compute exp_neg = exp(neg_part); then y = x + (alpha*(exp_neg - 1) - neg_part)? That's complicated. Better: compute two outputs:
  - out1 = x (for x>0)
  - out2 = alpha * (exp(x) - 1) (for x<=0)
  But we need to blend per element. Ascend C vector operations can use `Muls`, `Add`, `Exp` with masks. We can use `Compare` to create a mask tensor of 0/1, then use `Madd` or `Select`. Simpler: use `If` statement in scalar loop? But we need vectorized for performance. In Ascend C, we can use `Mul` with mask? There's `Madd` which does element-wise multiply-add with optional mask. But for branching, we can compute both expressions and then select based on mask. For example:
    - Compute mask = (x > 0) as float: 1.0 for true, 0.0 for false.
    - Compute neg_mask = 1.0 - mask.
    - Compute exp_all = exp(x) (for all, but exp(positive) is fine)
    - Then y = mask * x + neg_mask * alpha * (exp_all - 1.0).
  That works but exp(x) for large positive could overflow? In ELU, alpha is usually 1, but could be any float. However exp(x) for large positive is huge but it's multiplied by neg_mask=0 so doesn't matter. But still compute exp for all elements, which might be heavy but acceptable if we use vector exp. Alternatively compute exp only for negative part using `Mins` then exp of that. Let's do: compute neg_part = min(x, 0). Then compute exp_neg = exp(neg_part). Then y = x + (alpha*(exp_neg - 1) - neg_part)? Actually check: For x>0, neg_part=0, exp_neg=1, then alpha*(1-1)-0=0, y=x. For x<0, neg_part=x, exp_neg=exp(x), then alpha*(exp(x)-1) - x, so y = x + (alpha*(exp(x)-1) - x) = alpha*(exp(x)-1). Perfect! So formula: y = x + alpha*(exp(min(x,0)) - 1) - min(x,0). That uses only one exp call on the negative part, which is good. So we compute:
    - neg = mins(x, 0.0f)
    - exp_neg = exp(neg)
    - temp = alpha * (exp_neg - 1.0f)
    - y = x + temp - neg
  That's vectorizable.

We'll implement that in kernel.

Tiling: same as leaky_relu, with UB budget for floats: need space for x, y, neg, exp_neg, temp, maybe extra. In our kernel we need: inQueueX (tile), outQueueY (tile), tmp1 for neg, tmp2 for exp_neg, tmp3 for temp, maybe tmp4 for alpha*... We can reuse. Let's allocate three local tensors: one for neg, one for exp_neg, one for temp. That's 3*tile floats. Also need alpha constant. So total local tiles: inX, outY, neg, exp_neg, temp = 5 tiles. 5*tile floats. Same as leaky_relu (they used 5 tiles). So same UB budget calculation.

Tiling data: totalLength, tileLength, blockDim, alpha (float). We'll call `alpha` not `negativeSlope`.

Project json: op name EluCustom, input x (float), output y (float), attr alpha (float, default 1.0).

Kernel: function name `elu_custom`, class KernelEluCustom.

Python bind: similar, call aclnnEluCustom.
