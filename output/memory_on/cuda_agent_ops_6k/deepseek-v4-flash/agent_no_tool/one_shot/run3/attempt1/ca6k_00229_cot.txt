更高效的方法是：计算输出形状：N, C, H_out = H_in + padding[0] + padding[1], W_out = W_in + padding[2] + padding[3]。输出大小已知。kernel从全局输入读取，计算输出索引，如果索引在输入区域内，则复制值，否则写入0。

由于这是固定的padding，我们可以直接计算每个输出元素对应的输入索引。但AscendC的DataCopy通常用于连续内存块。我们可能需要对每个tile进行循环。对于4D张量，我们可以在内核中按N和C循环，然后按H和W进行tile处理。或者将张量展平为1D并计算偏移量？但这样会复杂。
