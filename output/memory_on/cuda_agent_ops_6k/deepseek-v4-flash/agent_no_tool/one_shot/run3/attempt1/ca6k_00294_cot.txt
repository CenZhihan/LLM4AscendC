我们被要求生成一个自定义AscendC算子，用于融合F.embedding操作。任务ID: ca6k_00294。参考程序使用nn.Embedding，输入是(batch_size, seq_length)的整数索引，输出是(batch_size, seq_length, embedding_dim)的浮点张量。

思路：我们需要实现一个fused算子，接收索引和权重矩阵，直接执行embedding lookup。由于AscendC的约束，我们通常需要处理内存访问模式。embedding lookup本质上是根据索引从权重矩阵中收集行。我们可以将其实现为gather操作，但更高效的是直接实现一个fused kernel。

在NPU上，可以将权重数据加载到UB中（如果embedding_dim不是太大），使用tiling按行处理。由于embedding_dim=2048，batch_size=64，seq_length=1024，总输出元素数=64*1024*2048=134,217,728个float，约512MB，这个很大。但我们可以逐块处理。

我们设计算子：接收输入索引（int64？实际上PyTorch中embedding输入是LongTensor，int64），权重矩阵（float32），输出（float32）。输入索引形状为(N,)，权重形状为(num_embeddings, embedding_dim)。输出形状为(N, embedding_dim)。

但这里存在padding_idx：如果索引等于padding_idx，输出应该全零。我们的kernel需要处理这个。

我们需要定义算子输入/输出：输入x (ND, int64)，weight (ND, float32)，输出y (ND, float32)。属性：padding_idx (int64, optional, default=-1?但这里Padding_idx=0)。也可以选择将padding_idx作为参数传入。

为了简化，我们可以把padding_idx作为属性传入。

在tiling中，我们需要计算总元素数：总行数（索引数量）* embedding_dim。我们按行处理，每tile加载若干索引和对应的权重行，然后写回。

由于每个输出行独立，我们可以使用block grid-stride pattern，每个block处理多个输出行。

但考虑到embedding_dim可能较大（2048），我们可以每tile处理固定行数，比如每个tile处理32行（每行2048个float，即8192字节）。UB大小大约192KB，可以容纳多个tile。

但为了简单，我们可以设计一个更直接的kernel：循环遍历所有输出行，每个线程（或者每个block）处理连续的一批行。使用tiling参数：totalRows（索引数量），tileRows（每批次处理的行数），embeddingDim，paddingIdx。

在kernel内部，我们可以使用DataCopy从全局内存（权重）加载一行数据到local tensor，并从输入索引中获取索引。注意input索引可能是int64，我们转换为int32使用（假设num_embeddings < 2^31）。然后检查是否为padding_idx，如果是则填充0。

```cpp
static ge::graphStatus InferShape(gert::InferShapeContext *context) {
    const gert::StorageShape *indices_shape = context->GetInputShape(0);
    const gert::StorageShape *weight_shape = context->GetInputShape(1);
    gert::StorageShape *y_shape = context->GetOutputShape(0);
    uint32_t indices_ndim = indices_shape->GetDimNum();
    uint32_t embedding_dim = weight_shape->GetDim(1); // 假设weight是2D
    gert::Shape out_shape(indices_ndim + 1); // 新形状维度+1
    for (uint32_t i = 0; i < indices_ndim; ++i) {
        out_shape.SetDim(i, indices_shape->GetDim(i));
    }
    out_shape.SetDim(indices_ndim, embedding_dim);
    y_shape->SetShape(out_shape);
    return ge::GRAPH_SUCCESS;
}
```

```cpp
static ge::graphStatus TilingFunc(gert::TilingContext *context) {
    Ca6k00294CustomTilingData tiling;
    const gert::StorageShape *indices_shape = context->GetInputShape(0);
    const gert::StorageShape *weight_shape = context->GetInputShape(1);
    uint32_t totalIndices = indices_shape->GetOriginShape().GetShapeSize(); // 总索引数量
    uint32_t embeddingDim = weight_shape->GetOriginShape().GetDim(1); // 第二维

    const gert::RuntimeAttrs *attrs = context->GetAttrs();
    const int64_t *paddingIdx = attrs->GetAttrPointer<int64_t>(0);

    // UB budget: 192KB, each row = embeddingDim * 4 + 8 bytes (but we use int32 for index? 还是int64)
    // 简化：每行权重：embeddingDim * 4 bytes, 索引8 bytes, 输出同样权重大小，双缓冲需要2倍
    // 每个tile: tileIndices * (2 * embeddingDim * 4 + 8) 字节。 取保守值
    uint32_t ubBytesPerTileRow = 2 * embeddingDim * 4 + 8; // 8 for int64 index, but we can reuse? 其实索引只需要一次，但为了简单，取上限。
    uint32_t maxTileIndices = 192 * 1024 / ubBytesPerTileRow;
    if (maxTileIndices > 64) maxTileIndices = 64; // 限制最大行数
    // 对齐到32
    maxTileIndices = (maxTileIndices / 32) * 32;
    if (maxTileIndices < 32) maxTileIndices = 32;

    uint32_t tileIndices = (totalIndices < maxTileIndices) ? totalIndices : maxTileIndices;
    // 确保不为0
    if (tileIndices == 0) tileIndices = 1;

    uint32_t blockDim = (totalIndices + tileIndices - 1) / tileIndices;
    if (blockDim < 8) blockDim = 8;
    if (blockDim > 24) blockDim = 24;

    context->SetBlockDim(blockDim);

    tiling.totalIndices = totalIndices;
    tiling.tileIndices = tileIndices;
    tiling.embeddingDim = embeddingDim;
    tiling.paddingIdx = (uint32_t)(*paddingIdx); // 转换为无符号
    tiling.blockDim = blockDim;

    tiling.SaveToBuffer(context->GetRawTilingData()->GetData(),
                        context->GetRawTilingData()->GetCapacity());
    context->GetRawTilingData()->SetDataSize(tiling.GetDataSize());

    size_t *currentWorkspace = context->GetWorkspaceSizes(1);
    currentWorkspace[0] = 0;
    return ge::GRAPH_SUCCESS;
}
```

kernel_src: 实现自定义embedding kernel。

思路：每个block负责处理tileIndices行，使用grid-stride循环。

输入：global buffer indices, weight, output。
每个block首先加载分配给自己的索引块，然后循环处理。
需要先加载索引数组（int64本地），然后对于每个索引，从权重中读取对应行（如果索引!=paddingIdx），否则写零。

由于Embedding查找是每个输出行独立，我们可以用简单的循环。

代码结构：

```cpp
#include "kernel_operator.h"

constexpr int32_t BUFFER_NUM = 2;

class KernelCa6k00294Custom {
public:
    __aicore__ inline KernelCa6k00294Custom() {}
    __aicore__ inline Init(GM_ADDR indices, GM_ADDR weight, GM_ADDR output,
                           uint32_t totalIndices, uint32_t tileIndices,
                           uint32_t embeddingDim, uint32_t paddingIdx)
    {
        totalIndices_ = totalIndices;
        tileIndices_ = tileIndices;
        embeddingDim_ = embeddingDim;
        paddingIdx_ = paddingIdx;

        indicesGm_.SetGlobalBuffer((__gm__ int64_t*)indices, totalIndices_);
        weightGm_.SetGlobalBuffer((__gm__ float*)weight); // 2D: (num_embeddings, embeddingDim)
        outputGm_.SetGlobalBuffer((__gm__ float*)output, totalIndices_ * embeddingDim_);

        pipe_.InitBuffer(inQueueIndices_, BUFFER_NUM, tileIndices_ * sizeof(int64_t));
        pipe_.InitBuffer(inQueueWeightRow_, BUFFER_NUM, tileIndices_ * embeddingDim_ * sizeof(float)); // 双缓冲重量数据
        pipe_.InitBuffer(outQueueY_, BUFFER_NUM, tileIndices_ * embeddingDim_ * sizeof(float));
    }

    __aicore__ inline void Process() {
        uint32_t blockNum = AscendC::GetBlockNum();
        uint32_t blockIdx = AscendC::GetBlockIdx();
        uint32_t strideRows = blockNum * tileIndices_;
        uint32_t startRow = blockIdx * tileIndices_;
        if (startRow >= totalIndices_) return;

        // 循环处理
        while (startRow < totalIndices_) {
            uint32_t curRows = tileIndices_;
            if (startRow + curRows > totalIndices_) curRows = totalIndices_ - startRow;

            // 加载索引块
            LoadIndices(startRow, curRows);

            // 加载权重行并计算
            Compute(startRow, curRows);

            startRow += strideRows;
        }
    }

private:
    __aicore__ inline void LoadIndices(uint32_t rowOffset, uint32_t rows) {
        AscendC::LocalTensor<int64_t> idxLocal = inQueueIndices_.AllocTensor<int64_t>();
        AscendC::DataCopy(idxLocal, indicesGm_[rowOffset], rows);
        inQueueIndices_.EnQue(idxLocal);
    }

    __aicore__ inline void Compute(uint32_t rowOffset, uint32_t rows) {
        AscendC::LocalTensor<int64_t> idxLocal = inQueueIndices_.DeQue<int64_t>();
        AscendC::LocalTensor<float> weightLocal = inQueueWeightRow_.AllocTensor<float>();
        AscendC::LocalTensor<float> outLocal = outQueueY_.AllocTensor<float>();

        // 对于每一行，根据索引加载权重或写零
        for (uint32_t r = 0; r < rows; ++r) {
```

由于需要多次DataCopy，可能会慢但可行。我们可以利用双缓冲流水线：两层循环，内层加载下一行权重，同时处理当前行。

设计如下：使用一个临时变量weightRowTmp（单行大小）和outRowTmp（单行大小）。对于每一行，先加载索引，然后如果索引有效，加载权重行，然后写输出。

但为了简化实现，我们使用同步方式：先加载所有索引到本地（tileIndices个int64），然后循环处理每行，对每行单独做DataCopy（权重行到本地）和DataCopy（本地到输出）。这不会流水线，但可以工作。由于数据量不大，可能可以通过。

- 分配inQueueIndices_ (BUFFER_NUM, tileIndices * sizeof(int64))
- 分配weightRowTmp_ (单行embeddingDim float)
- 分配outRowTmp_ (单行embeddingDim float)
- 加载索引
- 循环r从0到rows-1:
   idx = idxLocal[r]
   如果idx == paddingIdx: 写零到输出对应位置（使用DataCopy循环写零？可以使用SetZero并拷贝到输出）
   否则: 计算weight的偏移量：idx * embeddingDim，从weightGm_ DataCopy embeddingDim个float到weightRowTmp_，然后DataCopy从weightRowTmp_到输出对应位置
