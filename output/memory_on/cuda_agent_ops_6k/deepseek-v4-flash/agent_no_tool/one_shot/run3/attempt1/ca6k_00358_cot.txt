标准adaptive_max_pool2d行为：对于输出尺寸(H_out, W_out)，输入尺寸(H_in, W_in)，池化窗口大小stride = floor(H_in / H_out)？实际上是：对于每个输出位置，池化区域大小是ceil(H_in/H_out)和floor(H_in/H_out)？等等，PyTorch的adaptive_max_pool2d实现是：对于每个输出索引i，其对应的输入区域为 [i * (H_in / H_out) 到 (i+1) * (H_in / H_out)]，取最大值。具体实现是：对于3D输入（无batch），它计算每个输出位置对应的输入索引范围，可能有重叠或不规则。在自定义算子中，我们需要实现这个逻辑。

由于AscendC的向量化能力，我们可能直接写一个kernel，每个线程处理一个输出元素，计算对应的输入区域（可能很长）并求最大值。但为了效率，我们应该使用grid-stride循环。

在kernel中，我们需要从global memory读取输入数据。由于输入是NCHW，输出也是NCHW。对于每个输出索引(n, c, oh, ow)，我们需要计算输入区域范围。计算方式：
- h_start = floor(oh * H_in / H_out)
- h_end = ceil((oh+1) * H_in / H_out)  # 但注意不能超过H_in
- w_start = floor(ow * W_in / W_out)
- w_end = ceil((ow+1) * W_in / W_out)

然后遍历输入区域的所有(h,w)取最大值。

为了减少重复计算，我们可以将比例因子预先计算为浮点数。

但是AscendC的__aicore__函数中，我们可以使用标量float变量，并访问global memory。虽然效率不高，但对小规模输出（8192个输出元素）可能可行。

另一个问题是线程映射：我们使用blockIdx和blockDim来分配输出元素。每个block处理多个输出（grid-stride）。在每个block内，我们用循环处理分配给它的输出元素。

我们需要确定tile长度：每个tile处理多少个输出元素。由于输出总元素可能很大（假设N=16, C=3, 输出8192 => 393216，实际N*C=48, 输出元素=48*8192=393216）。这个数量适中，每个block处理几个到几十个元素。

我们可以在tiling中设置tileLength为每个block一次迭代处理的输出元素数量，比如1或2等。为了简化，可以设置tileLength=1，每个block每次迭代处理一个输出元素。然后使用grid-stride循环，直到处理完所有输出元素。

这样每个block的循环次数是 (totalOutputElems + blockNum -1)/blockNum。

blockNum可以通过tiling设置，比如24个block。

在kernel中，我们获取当前block的起始输出索引，然后循环tileLength次（一次处理tileLength个输出元素）。为了简化，我们可以设置tileLength=1，然后循环直到总元素处理完毕。

但是CopyIn和CopyOut使用DataCopy一次复制tileLength个输出元素的输入和输出。由于每个输出元素依赖于多个输入元素（池化窗口），我们不能简单复制一块输入。所以CopyIn不能使用基于输出元素大小的简单DataCopy。我们需要针对每个输出元素分别读取输入数据。因此，使用DataCopy批量复制输入行是不合适的。我们必须逐个访问输入元素。

所以，更合理的方案是：在kernel中，对于每个分配的输出元素，直接读取global memory中的输入数据（通过索引计算），计算最大值，然后写回输出。不使用local buffer做tile分批，因为输入窗口大小不定，难以组织成连续块。

我们仍然可以使用AscendC的DataCopy吗？可以，但需要为每个输出元素复制其窗口。如果窗口较小（如32x32），复制到local tensor再计算max是可以的，但复制开销加上计算可能比直接读global更慢？不过AscendC的global memory访问延迟较高，使用local buffer可以减少global访问。我们可以为每个输出元素分配一个local tensor来存放窗口数据，然后使用AscendC的MaxReduce（求向量最大值）来加速。但AscendC的MaxReduce是对向量操作，需要连续内存。窗口数据在输入中是连续的？对于NCHW，一个通道内的数据是连续的，窗口在一个通道内是连续的矩形区域，但跨行不是连续的（有行间距W）。要复制矩形区域，需要多行DataCopy，但每行是连续的，可以多次DataCopy。但这样会比较复杂。

在计算每个输出元素时：
- get n,c,oh,ow from outputIndex
- compute h_start, h_end, w_start, w_end
- float maxVal = -inf
- for h in [h_start, h_end):

        - val = xGm_[idx]

- yGm_[outputIndex] = maxVal
