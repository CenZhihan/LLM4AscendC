我们被要求生成一个自定义AscendC算子，替换PyTorch中的F.adaptive_max_pool1d_with_indices。参考程序是模型Model，接受输入x (batch, channels, length)，输出自适应最大池化1d的结果（以及indices，但模型只返回output）。我们只需要实现一个融合算子，输出与参考一致。

任务ID: ca6k_02116，运营名字使用ca6k_02116_custom和PascalCase Ca6k02116Custom。JSON项目描述需要包含op为主要字段。

约束：输出六个字符串变量：project_json_src, host_tiling_src, host_operator_src, kernel_src, python_bind_src, model_src。不能有eval_src。注意之前的修复memory：不要有OP_ADD重复注册；直接使用public字段赋值而不是setter；保持OpDef类和REG_OP；GetInputShape返回StorageShape*。另外，GetInputShape的转换问题允许使用GetDim。

我们考虑如何实现adaptive_max_pool1d_with_indices。给定输入(batch, channels, length)和输出大小output_size (一个标量)。自适应池化将输入长度分成output_size个区域，每个区域取最大值，并记录索引。需注意最后一个区域可能大小不同。通常实现：计算每个输出位置对应的输入窗口大小和偏移。对于任意长度的序列，池化窗口大小和步长可能不均匀。但AscendC实现要高效。

参考原始算子：F.adaptive_max_pool1d_with_indices。参数x形状(N, C, L)，output_size整数。输出形状(N, C, output_size)和indices(N, C, output_size)。我们只使用output。

融合算子输入：x (float)，以及标量attr: output_size (int32)，还有batch_size, channels, input_length (可能从形状推断)。tiling数据应包含这些信息以及池化参数。

我们设计一个自定义算子Ca6k02116Custom，输入x，输出y (output)。我们需要在算子内部实现自适应最大池化。在tiling时，计算每个块的处理量。

限制：需要保证数值精度匹配。

实现关键：在AscendC kernel中，对于每个(N, C)平面，遍历output的每个位置，计算输入窗口范围，取最大值。由于输入长度可能很大（比如100000），而输出大小256，所以每个输出位置窗口大小约L/output_size。我们可以逐元素处理，但效率可能低。但算力足够。

可考虑tile维度：将x按行分块，每个线程块处理一部分(N*C)平面和一部分output位置。

但简化起见，我们按输出元素粒度处理：每个线程块处理一定数量的输出元素（每个元素对应一个(N,C,out_pos)）。注意N=16, C=64, output_size=256，总输出元素16*64*256=262144，不算大。tile可按通道或批处理。

我们用一个简单的分块：输入x形状(N, C, L)，我们将其视为二维（batch * channels, L）。每个线程块处理一批连续的out_idx（比如64个），对于每个out_idx计算对应的窗口最大值。

由于每个输出元素的窗口范围可能不同，需要提前计算。在tiling中我们可以传递output_size和input_length，在kernel中动态计算窗口参数。

窗口计算：设输入长度L，输出长度O。对于输出索引i∈[0, O-1]，起始索引start = floor(i * L / O)，结束索引end = ceil((i+1) * L / O) - 1，窗口大小window = end - start + 1。然后在[start, end]范围内找最大值。索引从0开始。

注意数据类型是float，索引用int32。

我们需返回output，不需要indices。

在AscendC kernel中，我们可以使用LocalTensor，需要管理UB大小。根据示例，UB预算192KB。每个线程处理多个输出元素。假设每个输出元素需要局部存储读取一块输入（窗口大小最多ceil(L/O)+1）。当L=100000, O=256时，窗口最大为ceil(100000/256)=391，加上边界最多392。每个float 4字节，392*4=1568字节，非常小。我们可以一次处理多个输出元素，复用输入块？为了简单，我们给每个输出元素单独从全局读取窗口。由于窗口小，可以分批处理。

我们的tile设计：每个线程块处理一批输出元素（例如128个），每个元素计算窗口最大值需要读取输入的一个小窗口。我们可以使用DataCopy读取输入窗口到local tensor，然后计算最大值。由于窗口大小变化，我们可能需要对每个输出元素单独读取。为了简化，我们让每个线程处理一个输出元素，逐个处理。但这样并行度更高？由于输出元素总数约262k，我们可以用多个线程。

在tiling中，我们可以设置blockDim = min(20, total_output_elems / some_factor)。但为了通用，我们直接使用全局线程grid stride方式：每个线程处理一个输出元素（总输出元素= batch*channels*output_size）。使用grid stride循环处理多个元素。这样简单。

因此，在kernel中，每个线程处理一个输出元素（对应一个三维索引(n, c, out_idx)）。我们需要根据out_idx计算输入窗口范围。

但是AscendC编程模型通常使用DataCopy以块为单位，每个线程块处理一个tile。如果每个线程处理一个元素，数据移动可能不是最优。但鉴于每个线程窗口小，可以每个线程直接使用DataCopy从全局读取窗口（窗口大小不超过400），然后计算最大值，这是一个合法用法。但要注意AscendC运行时通常一个block对应一个内核实例，可以启动多个block（blockDim）。我们可以在kernel中使用GetBlockIdx()作为线程ID，使用grid stride for循环处理多个输出元素。

我们设tileLength为每个线程处理多少个输出元素？可以设为1，让每个线程处理一个，然后使用grid stride循环（步长=blockNum）。这样更加简化。

但是tiling中需要传递的参数：batch, channels, input_length, output_size。我们可以从输入形状获取，但需要反射。可以在host通过context获取输入形状维度得到batch, channels, input_length，然后传递给tiling数据。

在host_operator_src中，我们定义TilingFunc，获取输入形状，解析batch, channels, input_length，并设置tiling数据字段：batch, channels, input_length, output_size。同时计算blockDim。我们可以选择blockDim=24（最大）。每块处理输出元素数量：blockDim固定，每个线程处理多个输出元素，通过grid stride，所以可以设置tileLength = 1（每个线程处理的最小粒度），但实际上grid stride后每个线程处理多个。无需tileLength。更好是设置每个线程块处理固定的输出元素数，比如256。我们采用后者：每个block处理tileLength个输出元素，iterate over blocks. blockDim设为固定值，例如24。

```cpp
#include "ca6k_02116_custom_tiling.h"
#include "register/op_def_registry.h"

namespace optiling {

static ge::graphStatus TilingFunc(gert::TilingContext *context)
{
    Ca6k02116CustomTilingData tiling;

    const gert::StorageShape* shape = context->GetInputShape(0);
    uint32_t batch = static_cast<uint32_t>(shape->GetDim(0));
    uint32_t channels = static_cast<uint32_t>(shape->GetDim(1));
    uint32_t inputLength = static_cast<uint32_t>(shape->GetDim(2));

    const gert::RuntimeAttrs *attrs = context->GetAttrs();
    const int64_t *output_size_ptr = attrs->GetAttrPointer<int64_t>(0);
    uint32_t outputSize = static_cast<uint32_t>(*output_size_ptr);

    uint32_t totalOutputElems = batch * channels * outputSize;
    uint32_t blockDim = 24;
    uint32_t tileLength = (totalOutputElems + blockDim - 1) / blockDim;

    context->SetBlockDim(blockDim);

    tiling.batch = batch;
    tiling.channels = channels;
    tiling.inputLength = inputLength;
    tiling.outputSize = outputSize;
    tiling.totalOutputElems = totalOutputElems;
    tiling.tileLength = tileLength;
    tiling.blockDim = blockDim;

    tiling.SaveToBuffer(context->GetRawTilingData()->GetData(),
                        context->GetRawTilingData()->GetCapacity());
    context->GetRawTilingData()->SetDataSize(tiling.GetDataSize());

    size_t *currentWorkspace = context->GetWorkspaceSizes(1);
    currentWorkspace[0] = 0;
    return ge::GRAPH_SUCCESS;
}
```

```cpp
#include "kernel_operator.h"

class KernelCa6k02116Custom {
public:
    __aicore__ inline KernelCa6k02116Custom() {}

    __aicore__ inline void Init(GM_ADDR x, GM_ADDR y,
                                uint32_t batch, uint32_t channels,
                                uint32_t inputLength, uint32_t outputSize,
                                uint32_t totalOutputElems, uint32_t tileLength)
    {
        batch_ = batch;
        channels_ = channels;
        inputLength_ = inputLength;
        outputSize_ = outputSize;
        totalOutputElems_ = totalOutputElems;
        tileLength_ = tileLength;

        xGm_.SetGlobalBuffer((__gm__ float*)x, batch * channels * inputLength);
        yGm_.SetGlobalBuffer((__gm__ float*)y, totalOutputElems);
    }

    __aicore__ inline void Process()
    {
        uint32_t blockIdx = AscendC::GetBlockIdx();
        uint32_t blockNum = AscendC::GetBlockNum();

        uint32_t baseIdx = blockIdx * tileLength_;
        if (baseIdx >= totalOutputElems_) return;

        uint32_t count = tileLength_;
        if (baseIdx + count > totalOutputElems_) {
            count = totalOutputElems_ - baseIdx;
        }

        for (uint32_t i = 0; i < count; i++) {
            uint32_t globalIdx = baseIdx + i;
            // derive (n, c, out_idx)
            uint32_t tmp = globalIdx / outputSize_;
            uint32_t out_idx = globalIdx - tmp * outputSize_; // = globalIdx % outputSize_
            uint32_t n = tmp / channels_;
            uint32_t c = tmp - n * channels_;

            // compute window
            uint32_t start = out_idx * inputLength_ / outputSize_;
            uint32_t end = ((out_idx + 1) * inputLength_ + outputSize_ - 1) / outputSize_ - 1;
            if (end >= inputLength_) end = inputLength_ - 1; // safety

            uint32_t inputBase = (n * channels_ + c) * inputLength_ + start;
            float maxVal = -__FLT_MAX__;
            for (uint32_t j = start; j <= end; j++) {
                float val = xGm_[inputBase + (j - start)];
                if (val > maxVal) maxVal = val;
            }
            yGm_[globalIdx] = maxVal;
        }
    }

private:
    uint32_t batch_;
    uint32_t channels_;
    uint32_t inputLength_;
    uint32_t outputSize_;
    uint32_t totalOutputElems_;
    uint32_t tileLength_;
    AscendC::GlobalTensor<float> xGm_;
    AscendC::GlobalTensor<float> yGm_;
};
```
